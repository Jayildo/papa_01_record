# 노무비 모듈 제품 명세 초안

## 1. 목표

앱의 목표는 엑셀 양식을 그대로 재현하는 것이 아니라, 더 쉬운 입력 방식으로 동일하거나 더 신뢰할 수 있는 산출물을 만드는 것이다.

핵심 목표는 다음 네 가지다.

1. 근로자 정보를 입력, 저장, 수정할 수 있어야 한다.
2. 달력 형태의 근로일수/공수 입력을 바탕으로 노무비대장을 생성할 수 있어야 한다.
3. 입력된 데이터를 바탕으로 근로내역확인신고서를 자동 생성할 수 있어야 한다.
4. 노무비대장과 신고서에 공통으로 쓰이는 회사/현장 정보를 입력하는 `회사정보` 탭이 있어야 한다.

## 2. 상위 탭 구조

### 회사정보

노무비대장과 근로내용확인신고서에 공통으로 들어가는 정보를 입력한다.

### 근로자

근로자 기본정보를 저장, 수정, 조회한다.

### 노무입력

달력 형태로 일자별 근로일수 또는 공수를 입력한다.

### 노무비대장

입력된 근로 데이터와 계산 정책을 기준으로 공제, 차감지급액, 계좌정보, 월별 요약을 보여준다.

### 신고서

근로내용확인신고서를 근로자별 또는 기간별로 생성한다.

## 3. 회사정보 탭 항목

엑셀 `노무비대장`, `근로내용확인신고서`, `임금명세서10명`에 반복해서 등장하는 필드를 기준으로 한다.

### 필수

- 회사명
- 현장명
- 현장책임자명
- 작업연도
- 작업월
- 지급일

### 근로내용확인신고서용 후보

- 사업장관리번호
- 사업자등록번호
- 소재지
- 전화번호
- 대표자 또는 신고담당자 성명

위 항목은 엑셀 양식상 존재하며, 신고서 자동생성 시 사실상 필요하다.

## 4. 근로자 탭 항목

현재 엑셀 `근로자명` 시트를 기준으로 한다.

- 이름
- 주민번호
- 주소
- 연락처
- 일급
- 분야 또는 직종
- 팀명칭
- 은행명
- 계좌번호
- 예금주

추가 후보:

- 생년월일
- 국적
- 체류자격
- 고용유형
- 4대보험 적용유형

## 5. 노무입력 탭 구조

기본 입력 단위는 `근로자 x 일자`다.

### 입력 방식

- 월 달력형 그리드
- 값은 기본적으로 `0`, `0.5`, `1` 공수 단위
- 필요 시 추후 시간 입력 모드 확장 가능

### 부가 입력

- 비고
- 휴무/공휴일 표시
- 월별 총 공수 자동 계산

## 6. 노무비대장의 계산 철학

노무비대장의 핵심은 수식 자체보다 먼저 `적용 대상 판정`이다.

동일한 사람이라도 아래에 따라 계산 방식이 달라질 수 있다.

- 순수 일용근로자
- 계속고용이지만 일용 기준으로 관리되는 경우
- 국민연금/건강보험 적용 기준 충족자
- 고용보험만 적용되는 경우

따라서 계산 엔진은 아래 두 단계로 나눠야 한다.

1. 적용 판정
2. 금액 계산

## 7. 계산 정책표 초안

아래는 2026-04-08 기준으로 공식 자료를 바탕으로 정리한 초안이다.

### A. 일용근로소득세

- 대상: `일용근로자`로 판정된 경우
- 기본 공식:
  - 일별 과세표준 = `max(0, 일급 x 공수 - 150,000)`
  - 산출세액 = `일별 과세표준 x 6% x 45%`
- 해설:
  - 1일 15만원 공제는 소득세법상 일용근로자 공제 규정에 근거
  - `6% x 45%` 방식은 엑셀에 들어있는 계산과 방향이 일치
- 구현 상태:
  - 구현 후보 확정
  - 법적용 전제인 `일용근로자 판정 규칙`은 별도 확정 필요

### B. 지방소득세

- 대상: 일용근로소득세가 발생한 경우
- 기본 공식:
  - 지방소득세 = `소득세 x 10%`
- 구현 상태:
  - 구현 후보 확정

### C. 고용보험

- 대상: 일용근로자 포함 고용보험 적용 대상
- 기본 공식:
  - 근로자 부담분 = `보수 x 고용보험 근로자 요율`
- 2026-04-08 기준 참고값:
  - 근로자 부담 실업급여 요율 `0.9%`
- 구현 상태:
  - 요율은 구현 후보 확정
  - 적용 판정은 별도 확정 필요

### D. 국민연금

- 대상: 일용근로자 중에서도 계속고용 및 일수/시간 요건 등을 충족하는 경우만 적용 가능
- 2026-04-08 기준 참고값:
  - 총 보험료율 `9.5%`
  - 근로자 부담분 `4.75%`
- 주의:
  - 모든 일용근로자에게 일괄 공제하면 안 된다.
- 구현 상태:
  - 적용 조건 확정 전까지 자동 계산 보류

### E. 건강보험

- 대상: 직장가입자 적용 대상
- 2026-04-08 기준 참고값:
  - 총 보험료율 `7.19%`
  - 근로자 부담분 `3.595%`
- 주의:
  - 1개월 미만 일용근로자는 원칙적으로 직장가입 제외라서 일괄 공제하면 안 된다.
- 구현 상태:
  - 적용 조건 확정 전까지 자동 계산 보류

### F. 장기요양보험

- 대상: 건강보험 적용자에게 연동
- 2026-04-08 기준 참고값:
  - 총 보험료율 `0.9448%`
  - 건강보험료의 `13.14%`
- 주의:
  - 건강보험이 적용되지 않으면 장기요양보험도 자동 산정 대상이 아니다.
- 구현 상태:
  - 건강보험 정책 확정 후 종속 구현

## 8. 제품 정책 제안

법적용 오판을 피하기 위해 1차 구현은 아래처럼 가는 것이 안전하다.

### 1차

- 고용유형 선택 필드 도입
  - `일용`
  - `일용(4대보험 판정 필요)`
  - `일반근로`

- 자동 계산 범위
  - 일용근로소득세
  - 지방소득세
  - 고용보험

- 수동 또는 보류 항목
  - 국민연금
  - 건강보험
  - 장기요양보험

### 2차

- 국민연금/건강보험/장기요양보험의 적용요건을 월 기준 판정 로직으로 구현
- 각 항목별 계산 근거와 적용 사유를 UI에서 설명 가능하도록 구성

## 9. 설명 가능한 계산 UI 원칙

사용자가 각 공제 항목을 눌렀을 때 아래 정보를 봐야 한다.

- 적용 여부
- 적용 근거
- 사용한 입력값
- 계산 공식
- 최종 산출값

예:

- 일용근로소득세
  - 입력값: 일급 230,000원, 공수 1
  - 계산식: `(230,000 - 150,000) x 6% x 45%`
  - 결과: 2,160원
