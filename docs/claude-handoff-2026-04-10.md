## 1. 현재 상태 요약

이 프로젝트에는 기존 앱에 `노무비` 작업공간이 추가되어 있다.

현재 구현된 큰 축:
- `회사정보` 탭
- `근로자` 탭
- `노무입력` 탭
- `노무비대장` 탭
- `신고서` 탭
- Supabase 저장/조회 연동

현재 가장 중요한 상태:
- `근로내용확인신고서`는 더 이상 HTML 표 인쇄가 아니라 **공식 PDF 템플릿 오버레이 방식**으로 동작한다.
- `신고서` 탭 본문은 공식 PDF `iframe` 미리보기다.
- `신고서 인쇄`는 같은 PDF를 새 탭으로 연다.
- `노무비대장`은 여전히 HTML 문서 컴포넌트 기반이다.

## 6. 현재 문제 / 남은 작업

가장 중요한 미완료:

### 6-1. 신고서 PDF 좌표 정밀도
지금은 공식 PDF 위에 값이 올라가긴 하지만, **칸 위치가 완전히 확정된 상태는 아님**.

남은 작업:
- 상단 사업장 정보 줄 좌표 미세조정
- 각 근로자 블록 좌표 미세조정
- 일자 `○` 위치 미세조정
- 하단 서명부 좌표 미세조정

즉 기능은 동작하지만, 제출용 정밀도는 더 손봐야 함.

### 6-2. 공식 PDF 전수 대조의 부족
이번 세션에서 가장 큰 문제는 **공식 PDF의 모든 칸을 끝까지 전수 분석하기 전에 구현이 먼저 들어간 것**이었다.

따라서 Claude가 해야 할 우선 작업:
1. 공식 PDF를 실제 눈으로 대조
2. 각 칸이 어떤 앱 데이터와 매핑되는지 재점검
3. 누락 필드가 있으면 `회사정보` 또는 `근로자` 탭에 추가
4. `laborReportPdf.ts` 좌표를 다시 확정

### 6-3. 노무비대장
- HTML 문서 컴포넌트 기반
- 사용자 요청으로 날짜 헤더는 2단 표시 구조까지 일부 맞춤
- 하지만 법정서식이 아니라 “기존 회사 양식” 기준이므로 여전히 정밀도 조정 여지 있음

## 7. 사용자 피드백에서 중요했던 것

사용자는 다음에 매우 민감했다:
- 중간보고가 너무 잦은 것
- 실제로 안 끝난 상태에서 끝난 것처럼 말한 것
- 공식 PDF를 완전히 분석하지 않았는데 분석했다고 말한 것

따라서 다음 작업자는:
- 중간 경과보다 실제 결과 위주로 보고
- “완료” 표현을 매우 보수적으로 사용
- 공식 양식 전수 대조 후에만 완료라고 말하는 게 좋다
