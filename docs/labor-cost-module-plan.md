# 노무비 입력 웹앱 통합 계획

## 1. 기준 파일

- 엑셀 원본: `C:\project\nomu\26년1월노무비 (2).xlsx`
- 기존 웹앱: `C:\project\papa_01_record`

## 2. 엑셀 구조 요약

분석 결과 엑셀은 단일 입력표가 아니라 아래 흐름으로 구성된다.

1. `근로자명`
   근로자 마스터 입력 시트. 실제 수기 입력 필드는 이름, 주민번호, 주소, 연락처, 일급, 분야, 은행, 계좌다.

2. `신분증 안전 통장사본`
   마스터 입력값을 가져와 증빙 묶음 문서를 만든다.

3. `노무비대장`
   월별 현장 기준 집계 시트. 상단 메타정보는 회사명, 현장명 등이고 본문은 근로자별 일자 입력, 공수 합계, 노무비 총액, 보험/세금 공제, 차감지급액을 계산한다.

4. `근로내용확인신고서`
   근로자별 신고서 출력 시트.

5. `임금명세서10명`, `임금명세서10명 (2)`, `임금명세서10명 (3)`
   근로자 10명 단위 출력 시트. 지급항목, 공제항목, 실수령액을 표시한다.

## 3. 웹앱 정보 구조

엑셀 구조상 웹앱은 한 화면이 아니라 단계형 작업공간이어야 한다.

### 상위 모듈

- 기존 수목 계측 기능
- 신규 노무비 기능

### 노무비 서브탭

1. `근로자`
   근로자 마스터 관리

2. `노무입력`
   월/현장 기준 일자별 공수 입력

3. `정산/출력`
   근로자별 총 공수, 노무비, 공제, 차감지급액 요약

## 4. 권장 데이터 모델

초기 MVP에서는 아래 구조를 기준으로 한다.

### labor_project

- `id`
- `name`
- `company_name`
- `site_name`
- `work_year`
- `work_month`
- `manager_name`
- `payment_date`

### labor_worker

- `id`
- `project_id`
- `name`
- `resident_id`
- `birth_date`
- `address`
- `phone`
- `daily_wage`
- `job_type`
- `team_name`
- `bank_name`
- `account_number`
- `account_holder`
- `safety_doc_received`
- `id_card_received`
- `bankbook_copy_received`

### labor_entry

- `id`
- `project_id`
- `worker_id`
- `day`
- `units`
- `note`

`units` 는 0, 0.5, 1 같은 공수 단위를 기준으로 시작한다.

## 5. 계산 규칙 분리 원칙

엑셀 수식은 그대로 복사하지 않고 아래 규칙 함수로 재구현한다.

- 월 일수 계산
- 근로자별 총 공수
- 총 일수
- 총 지급액
- 공제 합계
- 차감지급액

세금/보험 공제식은 MVP에서는 수동 입력 또는 임시 0원 처리로 시작하고, 실제 실무 규칙 확인 후 고정한다.
